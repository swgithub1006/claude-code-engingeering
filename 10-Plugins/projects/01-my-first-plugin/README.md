# 示例插件：dev-helper

这是一个教学用的示例插件，展示插件的基本结构和创建方法。

## 插件内容

这个插件包含两类组件：

### 1. Slash Commands（斜杠命令）

| 命令 | 说明 |
|------|------|
| `/todo` | 快速添加 TODO 注释到代码 |
| `/explain` | 用简单语言解释选中的代码 |

### 2. Sub Agents（子代理）

| 子代理 | 说明 |
|--------|------|
| `quick-fix` | 快速修复小问题 |

## 目录结构

```
01-my-first-plugin/
├── plugin.json      # 插件元数据
├── commands/        # 斜杠命令
│   ├── todo.md
│   └── explain.md
└── agents/          # 子代理
    └── quick-fix.md
```

## 如何使用这个示例

### 方式一：作为本地插件

1. 将此目录作为插件源：
   ```
   /plugin install ./projects/01-my-first-plugin
   ```

2. 使用插件提供的命令：
   ```
   /todo 记得处理边界条件
   /explain 这段递归是怎么工作的？
   ```

### 方式二：学习参考

阅读各文件内容，理解：
- `plugin.json` 的格式
- 斜杠命令的定义方式
- 如何在插件中包含子代理

## 学习要点

1. **plugin.json 是必需的**：没有它，目录不会被识别为插件
2. **目录名即组件名**：`commands/` 下的文件变成斜杠命令
3. **复用子代理知识**：`agents/` 目录的格式与独立子代理完全相同
